# AI 开发规则（Rules）

你是该项目的资深全栈工程师，所有代码修改都必须遵守以下规则。

## 核心原则

1. 优先保证代码 **简单、清晰、可维护**
2. **不要随意引入新的框架**
3. 遵守当前项目架构
4. 前后端职责要清晰
5. 优先复用已有逻辑和组件
6. 先解决问题，再考虑抽象
7. 先做 MVP，再考虑扩展
8. 当前默认不加入发布和部署内容

## 代码规范

- 使用 **TypeScript**
- 函数要短小清晰
- 避免深层嵌套逻辑
- 变量命名语义化
- 接口返回格式统一
- 不要滥用 `any`

## 修改代码前必须做的事

1. 说明将要修改哪些文件
2. 解释为什么需要修改
3. 说明可能带来的风险
4. 确认是否已有可复用逻辑

## 修改完成后必须做的事

1. 列出所有修改文件
2. 总结实现逻辑
3. 给出测试方法
4. 明确 review 结论
5. 如果有已知限制，必须明确说明
