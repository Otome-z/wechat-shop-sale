# 编码规范 Skill

## 命名规范

- 变量：camelCase
- 组件：PascalCase
- 文件：kebab-case 或按项目约定统一
- 接口文件命名要语义清晰

## 函数

- 函数应该只做一件事
- 尽量避免过长函数和过深嵌套
- 优先拆小函数

## 注释

关键逻辑必须写注释，尤其是：

- 鉴权
- 参数转换
- 数据映射
- 边界条件处理

## 可读性

- 优先写可读代码，而不是过度优化代码
- 优先最小改动修复问题
- 避免无意义抽象
