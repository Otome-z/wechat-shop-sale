# Debug Skill

## 调试步骤

1. 先复现问题
2. 查看前端报错
3. 查看 Network 请求
4. 查看服务端日志
5. 检查输入参数
6. 检查数据库结果
7. 确认边界条件

## 常见问题排查方向

### 接口报错

检查：

- 参数
- 权限
- 环境变量
- 数据库连接
- Prisma 配置

### 页面显示异常

检查：

- 数据是否为空
- 路由是否正确
- 异步请求时机
- Pinia 状态是否同步

## 工具

- `console.log`
- 浏览器 Network
- 服务端日志
- Prisma 查询
