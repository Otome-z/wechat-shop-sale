# 开发流程（Workflow）

推荐开发顺序：

1. 明确需求边界
2. 先输出需求拆解
3. 再输出技术方案
4. 先做最小可用版本（MVP）
5. 先打通主链路
6. 再补安全、日志、鉴权、校验和体验
7. 最后做 review

## 默认流程

需求拆解
↓
技术设计
↓
开发
↓
验证
↓
Review

## 当前默认范围

包含：

- 微信小程序前端
- Node.js 后端
- MySQL / Prisma 数据层
- 基础工程化规范

不包含：

- 发布
- 部署
- 上线流程
- CI/CD
- 管理后台

## MVP 原则

第一版只实现最核心链路，不做过度设计。

适用于当前商城项目的 MVP：

- 登录
- 当前用户信息
- 商品列表
- 商品详情
- 购物车
- 提交订单
- 订单列表
- 订单详情

V1：

- 登录态持久化
- 参数校验
- 路由和页面访问控制
- 错误提示优化
- 操作日志

V2：

- 地址管理
- 优惠券
- 库存和订单状态优化
- 更多业务模块

## 任务拆分规则

每次只做一个小任务，例如：

- 接入登录接口
- 新增商品列表接口
- 新增订单表
- 优化购物车页面
- 修复某个下单 bug

不要在一个任务里同时做太多事情。

## 执行建议

前端任务优先顺序：

1. 页面结构
2. 状态管理
3. 接口联调
4. 交互优化

后端任务优先顺序：

1. 数据模型
2. 路由和控制器
3. Service 业务逻辑
4. 校验、鉴权、日志
